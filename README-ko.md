Artgine-Agent는 [Artgine](https://github.com/06fs4dix/Artgine) 엔진 위에서 동작하는 웹 기반 에이전트 인터페이스입니다. 파일 브라우저와 AI 어시스턴트를 하나의 앱으로 제공하며, Artgine 데스크톱 또는 웹 서버를 통해 실행됩니다.

> Language **[English](README.md)**

## 기능

### 파일 탭 `F3`
서버의 파일을 탐색하고 관리합니다.

- 폴더 이동, 이미지 보기, 오디오/비디오 재생
- `.ts` `.js` `.html` `.json` `.txt` 파일을 Monaco 에디터로 편집
- 편집한 파일 서버에 업로드/저장

### AI 탭 `F4`
iframe 기반 패널에서 AI 채팅과 터미널 세션을 관리합니다.

**Chat (채팅 세션)**
- **새 채팅 세션 시작 (`+ New Chat`)**: 모달창을 통해 아래의 옵션을 세팅하여 새로운 AI 에이전트 채팅 세션을 생성합니다.
  - *Working Directory*: 에이전트의 루트 작업 경로를 임의 지정합니다.
  - *Allow working dir write*: 에이전트에게 지정된 작업 경로 내 파일 쓰기/수정 권한을 부여합니다.
  - *MCP*: Model Context Protocol 툴 연동 허용 여부를 설정합니다.
  - *Copy MD*: 마크다운 서식 복사 지원 여부를 설정합니다.
- 다중 AI 프로바이더 지원: Claude, Codex, Antigravity
- **세션 상태 표시 컬러**: 🔴 빨간색(미연결/off), 🟡 노란색(처리 중/busy), 🟢 초록색(대기 중/idle)
- **세션 활성화 컬러**: 현재 선택되어 활성화된 채팅 세션은 목록에서 **옅은 파란색 배경**으로 강조됩니다.
- 각 세션별 공유 링크 생성 및 세션 삭제 지원
- 브라우저 탭이 포커스를 잃은 상태에서도 에이전트 작업 완료 시 사운드 및 시스템 알림 연동

**Terminal (터미널 세션)**
- **새 터미널 세션 시작 (`+ New Terminal`)**: 새로운 터미널 세션을 구성하고 실행합니다. (동시 최대 9개 세션 제한)
  - *구동 모드*: `cmd`(일반 명령 프롬프트), `claude`, `codex`, `antigravity(agy)` (AI 쉘 연동) 중 선택합니다.
  - *Key*: 다중 터미널을 직관적으로 구분하기 위한 세션 별명을 지정합니다. (선택사항)
  - *Working Directory*: 터미널 실행 시의 최초 시작 폴더 경로를 지정합니다.
  - *Allow working dir write*: 터미널 에이전트에게 파일 직접 수정/생성 권한을 부여합니다. (`cmd` 모드 시 비활성화)
  - *MCP / Copy MD*: MCP 툴 체인 활용 여부 및 마크다운 복사 권한을 설정합니다.
- **세션 상태 표시 컬러**: 🔴 빨간색(미연결/off), 🟡 노란색(처리 중/busy 또는 권한 승인 대기/wait), 🟢 초록색(대기 중/idle)
- **세션 활성화 컬러**: 현재 선택되어 활성화된 터미널 세션은 목록에서 **옅은 초록색 배경**으로 강조됩니다.
- 각 터미널 세션별 공유 링크 제공, 새 창(Window) 또는 모달 창으로의 독립적 실행(팝업) 기능 지원

**Schedule (반복 스케줄러)**
- **새 스케줄 등록 (`+ New Schedule`)**: 터미널과 연동하여 특정 주기마다 특정 명령을 백그라운드에서 자동 실행합니다.
  - *Name*: 등록할 스케줄의 식별 고유 이름(키)입니다.
  - *Terminal Key*: 명령을 실행할 대상 터미널 세션의 고유 Key를 지정합니다.
  - *Mode*: `none`, `cmd`, `claude`, `codex`, `antigravity(agy)` 중 지정하며, 대상 터미널 세션이 부재할 시 해당 모드의 터미널을 백엔드에서 자동 개설한 후 실행합니다.
  - *Delay (sec)*: 명령어를 반복 실행할 주기(초 단위)를 설정합니다. (최소 1초)
  - *Count*: 총 반복 실행할 횟수를 지정합니다. (`0` 입력 시 수동 삭제 전까지 무한 실행)
  - *Start / End offset (sec)*: 스케줄 시작까지의 최초 대기 지연 시간 및 지정 시간 경과 후 스케줄 자동 삭제 만료 시간을 설정합니다.
  - *Command*: 대상 터미널에서 실행할 쉘 명령어 혹은 프롬프트 텍스트를 입력합니다. (예: `node backup.js`)
  - *Advanced*: 스케줄 구동 환경에 대한 디테일한 작업 디렉토리 지정, 쓰기 권한, MCP, Copy MD 활성화 여부를 설정합니다.
- **스케줄러 모드 뱃지 컬러**: 스케줄러 모드(Mode)에 따라 목록 뱃지의 색상이 구분되어 표시됩니다.
  - 회색 (`none`) / 하늘색 (`cmd`) / 노란색 (`claude`) / 파란색 (`codex`) / 빨간색 (`antigravity/agy`)

### 단축키 및 화면 제어

#### 1. 전역 단축키 (어느 화면에서나 동작)
| 키 | 동작 |
|----|------|
| `F2` | 파일 검색 창 열기 |
| `F3` | 파일 브라우저 탭으로 전환 및 루트 폴더(`/`) 이동 |
| `F4` | AI 어시스턴트 탭으로 전환 |

#### 2. AI & 터미널 화면용 단축키
| 키 | 동작 | 조건/상황 |
|----|------|-----------|
| `Tab` | 채팅/터미널 세션 목록이 나오는 왼쪽 사이드바 열기/닫기 토글 | AI 패널 활성화 상태 |
| `ArrowUp`/`ArrowDown` | 세션 목록(채팅 방 / 터미널 창) 위/아래 이동 및 전환 | AI 사이드바가 열려 있을 때 |
| `ArrowRight` | AI 작업 완료 알림 팝업 시 해당 세션으로 즉시 화면 전환 | 알림 팝업 활성화 상태 |
| `ArrowLeft` | 알림 세션으로 이동한 직후 원래 보던 세션으로 복귀 (8초 제한) | 복귀할 이전 세션 존재 시 |
| `Shift + N` | 새 터미널 세션 (`cmd`) 즉시 생성 | 터미널 탭 및 사이드바 활성화 상태 |
| `Shift + D` | 현재 켜진 터미널 세션 종료 및 삭제 (컨펌 모달) | 터미널 탭 및 사이드바 활성화 상태 |

#### 3. 터미널 창 전용 단축키 & 입력 바 조작
| 키 | 동작 |
|----|------|
| `F6` | **SUPER 모드 토글** (에이전트가 실행하는 모든 명령의 자동 승인 모드 ON/OFF, 활성화 시 **입력창 테두리가 빨간색**으로 변함) |
| `F7` | 터미널 하단 입력 바(Input Bar) 보이기 & 텍스트 포커스 |
| `Ctrl + T` | 터미널 스크롤 최하단으로 이동 |
| `Enter` | 하단 입력 바에서 작성한 텍스트 전송 (빈 창일 경우 줄바꿈 전송) |
| `Arrow 키` | 입력 바가 완전히 비어있을 때 터미널 내부 커서 이동 명령 전송 |

> 💡 **터미널 입력 바 자동완성**:
> 입력 바에 `/`를 입력하면 에이전트 내장 명령어 및 커스텀 스킬 목록이 팝업됩니다. **`ArrowUp` / `ArrowDown`**으로 선택을 조정하고 **`Tab`** 또는 **`Enter`**로 적용할 수 있으며, **`Escape`**로 목록을 닫습니다. (입력 바에 직접 `/super` 입력 시 SUPER 모드 토글)

> 💡 **터미널 입력 바 상태 테두리 컬러**:
> 입력 바에 포커스가 잡혔을 때 현재 모드에 따라 테두리 및 배경 색상이 다르게 표시됩니다.
> - **일반 모드**: 포커스 시 **청록색(Emerald Green)** 테두리로 표시됩니다.
> - **SUPER 모드**: 테두리가 **빨간색(Red)**으로 변하고 입력창 배경이 붉은색 톤으로 물들어 자동 승인 모드 상태임을 경고합니다.

#### 4. 터미널 우측 드롭다운 메뉴 (`⌨` 버튼)
하단 입력 바 오른쪽에 위치한 `⌨` 버튼을 클릭하면 마우스/터치 조작용 가상 제어판이 활성화됩니다.
* **View**: 스크롤 하단 이동, 창 크기에 맞춘 화면 크기 재조정, `Ctrl+T` 전송, 입력 바 숨기기
* **가상 키**: 방향키(`↑`, `↓`, `←`, `→`), `Enter`, `ESC` 키 신호 전송
* **SUPER**: 자동 승인 모드 켜기/끄기
* **Command**: 에이전트 제어 명령어 단축 전송 (`RESUME`, `MODEL`, `COMPACT`, `CLEAR`, `STATUS`, `EXIT`)
* **Skill**: 사전에 등록한 커스텀 매크로 스킬 불러오기

> 💡 **단축키 전파**: 터미널 및 Monaco 파일 편집기 등 하위 프레임(iframe)에 마우스 포커스가 간 상태에서도 부모 창에 단축키 이벤트가 전달되어 동일하게 탭을 전환하거나 사이드바를 제어할 수 있습니다.

## AI 폴더 (ai/)

프로젝트 루트의 `ai/` 폴더는 AI 에이전트의 구동 환경 설정, 자동 권한 제어, 문법 검사 스크립트, 개별 작업 공간 및 마크다운 개발 가이드 문서를 관리합니다.

* **`settings.json`**: AI 에이전트의 권한(Permissions) 자동 처리 설정 파일입니다.
  - 읽기(`read`), 쓰기(`write`), 셸 실행(`run_command`/`Bash`) 툴에 대해 자동으로 실행을 승인(`allow`)하거나 거부(`deny`)할 필터 규칙을 지정합니다.
  - 최상위 `"log": true` 설정을 통해 에이전트 대화 로그 저장 여부를 결정합니다.
* **`tsc_check.js`**: TypeScript 파일 수정 후 문법 및 타입 정적 체크를 담당하는 스크립트입니다. 수정 사항 보고 전에 실행하여 정적 오류를 미리 방지할 수 있습니다.
* **가이드 마크다운 (`.md`) 파일들**: AI 에이전트가 코드 구현 시 의무적으로 준수해야 하는 각종 규칙 지침 문서들입니다.
  - [CodeNamingGuide.md](file:///E:/svn/Artgine/WebContent/ai/CodeNamingGuide.md): 코드 설계 규칙 및 명명 규칙 가이드
  - [ProjectSetup.md](file:///E:/svn/Artgine/WebContent/ai/ProjectSetup.md): 신규 개발 프로젝트 추가 및 기초 셋업 가이드
  - [UIGuide.md](file:///E:/svn/Artgine/WebContent/ai/UIGuide.md): 내부 프레임워크 기반 UI 개발 가이드
  - `ROLE.md` (혹은 `AGENTS.md`): AI 에이전트의 핵심 행동 원칙과 역할 지침
* **`workspace/`**: 에이전트가 각 세션별로 코드를 안전하게 테스트하거나 개발을 진행할 수 있도록 독립된 임시 작업 환경(Workspaces)이 실시간으로 격리 생성되는 디렉토리입니다.
* **`skill/`**: AI 에이전트가 단축 퀵 명령어로 사용할 수 있는 사전 정의된 스킬(매크로 명령 세트)들이 저장되는 폴더입니다.
* **`ctx/`**: AI 에이전트의 실행 세션 대화 컨텍스트 및 JSON 로그들이 누적되는 폴더입니다.

## 서버

Artgine 서버에서 다음 라우터로 동작합니다.

| 라우터 | 역할 |
|--------|------|
| `CFileServer` | 파일 브라우저 및 업로드 |
| `COAuthServer` | 패스워드 인증 |
| `CTerminalRouter` | 터미널 세션 관리 (ttyd) |
| `CAIChatRouter` | AI 채팅 WebSocket 및 REST API |

## 시작하기

```bash
git clone --recursive https://github.com/06fs4dix/Artgine-Agent.git
cd Artgine-Agent
npm install
npm start
```

Artgine 앱에서 `projectPath`를 `proj/Home`으로 설정하고 **Run**을 클릭하세요.

> **웹 서버만 실행** — `npm run start_web [포트]` (기본값: 8050)
